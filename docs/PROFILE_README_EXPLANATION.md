# Profile README Explanation

This document explains the structure and writing choices behind the root `README.md`.

## 1. Positioning

The README opens with:

- Information and Computing Science undergraduate
- mathematics background
- AI Engineering
- Backend Development
- LLM Applications
- Document Intelligence

This gives HR a quick role match while also giving technical interviewers a clear direction for follow-up questions.

## 2. English-First Style

The README is mostly English because it needs to work for:

- large-company technical recruiting
- overseas project applications
- technical interviewers who scan GitHub quickly
- bilingual resume / portfolio usage

Chinese is kept only in the name line so the identity remains clear.

## 3. Project Status Boundaries

The README separates three types of evidence:

- **Public repositories:** AI Document Q&A Assistant, Python Workflow Automation Workbench, Vue Admin Dashboard Practice.
- **Project experience / system design:** AI Job Application Agent.
- **Academic research:** Document Intelligence and 5G Network Slicing.

This avoids overclaiming. For example, the AI Job Application Agent is described as project experience and system design, with public implementation still being organized. The architecture stack is included, but the README does not claim production deployment or enterprise usage.

## 4. Why These Sections

### Opening Intro

The opening intro is short and role-focused. It avoids generic phrases such as "passionate about technology" and instead states what Wendy actually builds: structured, reviewable, and testable AI workflows.

### Current Focus

This section tells HR and interviewers what to remember after 30 seconds:

- LLM workflows
- backend systems
- document intelligence
- engineering fundamentals

### Tech Stack

The stack is grouped by hiring signals:

- languages
- backend
- AI / ML
- data / storage
- tools
- frontend

This is easier to scan than a long unstructured list.

### Featured Projects

The order is intentional:

1. AI Job Application Agent
2. Document Intelligence
3. 5G Network Slicing
4. AI Document Q&A Assistant
5. Python Workflow Automation Workbench
6. Vue Admin Dashboard Practice

The first three match the target direction and required project background. The last three provide public engineering evidence.

### What I'm Building

This section makes the profile feel active without sounding exaggerated. It also explains why some projects are currently briefs or prototypes instead of complete public systems.

### Contact

Email, portfolio, and GitHub are included plainly. No extra visual noise is added.

## 5. What Was Intentionally Avoided

- No exaggerated claims such as "production-grade", "enterprise deployed", or "state-of-the-art".
- No claim that research projects are public software repositories.
- No claim that the AI Job Application Agent is already a complete public implementation.
- No heavy animation, decorative badges, or oversized GitHub stats.
- No motivational filler or broad claims like "passionate about everything".

## 6. Optional GitHub Stats

GitHub stats were not added by default because they can distract from project evidence. If added later, they should appear below the project sections, not above them.
