## Principles

- Prefer 2-4 meaningful commits per week across the portfolio.
- Each commit should improve code, documentation, experiments, tests, or evaluation.
- Avoid large batches of unrelated changes.
- Keep commit messages clear and professional.
- Use experiment results and documentation updates as visible evidence of progress.

```text

exp(recsys): add feature ablation report
docs(agent): document verifier failure cases
```
