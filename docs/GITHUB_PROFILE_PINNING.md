# GitHub Profile Pinning Guide

Use this guide to keep the GitHub first impression aligned with recommendation/search/retrieval algorithm roles.

## Pin These 6 Repositories

Pin these repositories in this order:

1. `short-video-recsys-reproduce`
2. `ecommerce-multimodal-retrieval`
3. `llm-rag-system`
4. `recommendation-system`
5. `transformer-model-lab`
6. `open-source-reading-notes`

## Keep Out Of The First Screen

Do not pin these if applying to algorithm internships:

- frontend practice sites
- static ecommerce pages
- Vue admin dashboards
- course templates
- generic portfolio/demo pages
- document-QA projects that are not tied to retrieval evaluation

They do not need to be deleted, but they should not compete with the recommendation/search/retrieval evidence chain.

## Suggested Manual Steps

1. Open `https://github.com/Wendy-James`.
2. Click `Customize your pins`.
3. Remove frontend/static/course/admin practice repositories from the pinned list.
4. Add the 6 repositories listed above.
5. Save the profile pins.
6. If a practice repository is outdated and not useful for applications, consider archiving it from `Settings -> Danger Zone -> Archive this repository`.

## Interview Boundary

If asked why other repositories still exist:

> I keep older frontend and course-practice repositories public as learning records, but they are not my current application story. For algorithm roles, the relevant evidence is the pinned recommendation/search/retrieval line: short-video recommendation, ecommerce multimodal retrieval, and RAG retrieval evaluation.
